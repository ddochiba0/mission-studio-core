# Sprint 5 — Completion rules and participant preview

## Goal

Define reusable completion and reward rules and let an author preview the participant experience.

## Included

- Completion Engine isolated from applications
- complete-all or minimum-count policies
- optional ordered checkpoint enforcement
- optional completion reward metadata
- interactive participant preview and progress bar
- automatic browser storage migration from schema V1 to V2

## User verification

1. Configure a mission to require all checkpoints in order.
2. In participant preview, select the second checkpoint first and confirm the warning.
3. Complete checkpoints in order and confirm mission completion.
4. Enable a reward and confirm its title appears after completion.
